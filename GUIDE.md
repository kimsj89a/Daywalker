# Daywalker - 팀 프로젝트 일정 관리 도구

## 시작하기

### 1. 파일 다운로드
공유된 Google Drive에서 `Workflow.html` 파일을 다운로드하세요.

### 2. 브라우저에서 열기
다운로드한 `Workflow.html`을 더블클릭하면 브라우저에서 바로 실행됩니다.
(Chrome, Edge, Firefox 모두 지원)

### 3. 동기화 확인
사이드바 하단에 **"● 실시간 동기화 중"** 이 초록색으로 표시되면 팀 데이터가 연결된 상태입니다.

| 상태 | 의미 |
|------|------|
| ● 실시간 동기화 중 (초록) | 정상 연결. 모든 변경이 팀과 실시간 공유됨 |
| ● 저장 중... (노랑) | 데이터 저장 진행 중 |
| ● 오프라인 (빨강) | 인터넷 연결 안 됨. 로컬에서만 동작 |

## 주요 기능

### 프로젝트 관리

- **새 프로젝트 만들기**: 사이드바 `+ 새 프로젝트` 버튼 클릭
- **그룹 만들기**: `📁 새 그룹` 버튼으로 프로젝트를 폴더처럼 분류
- **프로젝트 순서 변경**: 사이드바에서 프로젝트를 드래그하여 순서 변경
- **프로젝트 삭제**: 사이드바에서 프로젝트를 🗑️ 휴지통으로 드래그

### 태스크 관리

- **태스크 추가**: 프로젝트 선택 후 `+ 태스크 추가` 버튼
- **태스크 편집**: 태스크 바를 클릭하면 편집 모달 열림
- **일정 변경**: 간트 차트에서 태스크 바를 드래그하여 날짜 이동
- **기간 조절**: 태스크 바의 오른쪽 끝을 드래그하여 기간 조절
- **완료 처리**: 태스크 우클릭 → `완료/미완료 토글`
- **태스크 복제**: 태스크 우클릭 → `복제`
- **태스크 이동**: 사이드바의 다른 프로젝트로 태스크를 드래그하면 프로젝트 간 이동

### 의존성 (태스크 연결)

- 태스크 편집 시 `의존 태스크`를 선택하면 태스크 간 화살표 연결선이 표시됩니다.
- 선행 태스크가 끝난 뒤 후속 태스크가 시작되는 관계를 시각화합니다.

### 메모

- 프로젝트별로 메모를 작성할 수 있습니다.
- `📝 메모` 버튼 클릭 → 메모 패널 열림
- 회의록, 참고사항 등을 기록하는 데 활용하세요.

### 대시보드

- 사이드바 `📊 대시보드` 클릭 시 전체 프로젝트 현황을 한눈에 확인
- 프로젝트별 진행률, 태스크 수, 마감일 등 요약 표시

## 간트 차트 조작

| 조작 | 방법 |
|------|------|
| 확대/축소 | 우측 상단 `−` / `+` 버튼 |
| 태스크 이동 | 간트 바를 좌우로 드래그 |
| 기간 조절 | 간트 바 오른쪽 끝을 드래그 |
| 태스크 순서 변경 | 태스크 목록에서 행을 드래그 |
| 이름/설명 인라인 편집 | 프로젝트 이름이나 설명을 클릭 |

## 데이터 백업 & 복원

- **내보내기**: 사이드바 `💾 데이터 내보내기` → JSON 파일로 다운로드
- **가져오기**: 사이드바 `📂 데이터 가져오기` → JSON 파일 선택하여 복원
- **CSV 내보내기**: 프로젝트 내에서 `📊 CSV 내보내기` → 태스크 목록을 CSV로 다운로드

> 중요한 마일스톤 전에는 데이터를 내보내기해서 백업해두는 것을 권장합니다.

## 실시간 동기화 안내

- 팀원 A가 태스크를 추가/수정하면 팀원 B의 화면에 **즉시 반영**됩니다.
- 각자의 **현재 보고 있는 프로젝트**, **그룹 접기/펼치기** 상태는 개인별로 유지됩니다.
- 인터넷이 끊기면 로컬 모드로 전환되며, 재연결 시 자동 동기화됩니다.

## 자주 묻는 질문

**Q: 데이터는 어디에 저장되나요?**
A: Firebase Realtime Database (Google 클라우드)에 저장됩니다. 브라우저를 닫아도 데이터는 유지됩니다.

**Q: 동시에 같은 태스크를 수정하면 어떻게 되나요?**
A: 마지막으로 저장한 내용이 반영됩니다. 가능하면 같은 태스크를 동시에 수정하는 것은 피해주세요.

**Q: 오프라인에서 수정한 내용은 어떻게 되나요?**
A: 오프라인 중에는 로컬에만 저장됩니다. 인터넷이 다시 연결되면 Firebase와 동기화됩니다.

**Q: 파일을 새 버전으로 교체하면 데이터가 사라지나요?**
A: 아닙니다. 데이터는 Firebase에 저장되므로 HTML 파일을 교체해도 데이터는 그대로 유지됩니다.

**Q: 다른 팀과 데이터를 분리하고 싶어요.**
A: HTML 파일 상단의 `WORKSPACE_ID` 값을 팀별로 다르게 설정하면 됩니다. (예: `'team-alpha'`, `'team-beta'`)
